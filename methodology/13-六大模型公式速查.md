# 13 · 后 6 大模型公式速查（Runway · Pika · Hailuo · Hunyuan · Wan · 即梦）

> 本仓库收录的 11 个模型分两组：**前 5 个**（Seedance / HappyHorse / Kling / Sora / Veo）各有独立的方法论篇章；**后 6 个**因为公式高度相似（基础写法 + 模型特色）合并在此速查表。

## 整体对照

| 模型 | 厂商 | 核心公式 | 特色能力 |
|---|---|---|---|
| **Runway Gen-4 / Aleph** | Runway | `subject + action + setting + camera + motion + style + constraints` | Aleph 视频编辑（add / remove / change / re-light / re-style）|
| **Pika 2.5** | Pika Labs | `subject + action + setting + Pikaffect/style + camera` | 15+ 创意特效（Melt / Explode / Squish 等）+ Pikaframes 关键帧 |
| **Hailuo 02** | MiniMax | "Director's AI"，**克制简洁**，重视物理细节 | 物理仿真业界第一（水/火/烟/布料/重力）|
| **Hunyuan Video 1.5** | 腾讯 | Normal/Master 双模式 prompt rewrite | 开源最强 + 跨维度生成 + LoRA 训练 |
| **Wan 2.5** | 阿里巴巴 | `Entity + Scene + Motion + Sound` | 原生音频 + 数字人 lip-sync 最准 |
| **即梦 3.0** | 字节剪映 | 8 维度公式（中文友好） | 中文最强 + 剪映集成 + 百款模板 |

## 1. Runway Gen-4 / Aleph

### Gen-4 文生视频公式

```
[Subject — specific details] [Action — clear verbs] [Setting]
[Camera — framing + movement] [Motion — how it evolves over time]
[Style — film stock / mood / era] [Constraints — no morphing etc.]
```

### Aleph 编辑公式（核心动词）

| 动词 | 用途 | 示例 |
|---|---|---|
| `add` | 添加元素 | "Add streetlights to make this look like night" |
| `remove` | 移除元素 | "Remove all background pedestrians" |
| `change` | 改变元素 | "Change the camera to a wider angle" |
| `replace` | 替换元素 | "Replace the car with a vintage motorcycle" |
| `re-light` | 重打光 | "Re-light with colder blue tones" |
| `re-style` | 换风格 | "Re-style as hand-painted watercolor animation" |

### 写 Aleph 的 3 原则

1. **说什么要改**（change something）
2. **说什么要保持稳定**（preserve everything else）
3. **说转换的程度**（how much / how natural）

样例：[rw-001](../prompts/seedance/README.md) 到 rw-012 均为官方推荐结构。

## 2. Pika 2.5

### 标准模板

```
A [subject] [action] in [setting], [style/lighting], [camera], [quality], [no morphing].
```

### Pikaffects 完整列表

| 类别 | 效果 |
|---|---|
| 形变类 | Melt / Inflate / Deflate / Squish / Crumble / Tear |
| 爆裂类 | Explode / Crush / Eye-pop |
| 物理类 | Levitate / Poke / Dissolve / Peel |
| 替换类 | Cake-ify（任何物品变蛋糕）|
| 出场类 | Ta-da |

### Pikaframes 关键帧延长

定义首帧 + 尾帧，模型生成中间过渡：

```
Start: A single cherry blossom tree in full bloom in spring sunlight.
End: The same tree with golden autumn leaves and falling foliage.
Smooth morphing transition over time.
```

### 5 条铁律

1. **单一主体**：别塞多个角色
2. **小步迭代**：一次只改一个变量
3. **明确风格**：每个 prompt 选一个美学锚点
4. **加 "no morphing"**：避免形变畸形
5. **写 "1080p" 或 "high quality"**：默认输出会偏低质

## 3. Hailuo 02

### Director's AI 思维

> Hailuo operates as a "Director's AI" that wants a script.

它喜欢叙事流和时间关系，不喜欢被堆砌的视觉细节淹没。

### 公式（克制版）

```
[1-2 个主体清晰描述] [清晰的时序动作] [简洁场景] [物理细节强调]
```

### 物理关键词清单

| 类型 | 关键词 |
|---|---|
| 流体 | water spray / fluid dynamics / surface tension |
| 火焰 | flame flickers / smoke trails / combustion |
| 布料 | fabric folds / weight / silk ripples |
| 重力 | angular momentum / accurate gravity / knee absorption |
| 材质 | wood shavings / wax pooling / Maillard reaction |

### 反例 vs 正例

❌ "A beautiful epic ocean scene with dramatic waves and many cool effects."
✅ "Massive ocean waves crash against rugged cliff rocks at high speed, water spray erupts upward and disperses in mist, foam pools form and drain in cycles."

样例：[hl-001](../prompts/seedance/README.md) ~ hl-012 全部突出物理感。

## 4. Hunyuan Video 1.5

### 双模式 Prompt Rewrite

| 模式 | 用途 | 适合场景 |
|---|---|---|
| **Normal** | 理解用户意图 | 简单场景、基础测试 |
| **Master** | 增强构图/光照/运镜描述 | 复杂电影感、高质量输出 |

### 图生视频铁律（同 Kling）

**只描述运动，不重描述图中已有内容**：

❌ "A woman in red dress in front of a window..."（重描述）
✅ "The wind picks up; her hair lifts and floats around her face, she slowly turns toward the camera."

### 独有：跨维度生成

虚拟（2D/3D/动漫）角色无缝融入真实场景。例：[hy-001](../prompts/seedance/README.md) 3D 动漫角色走过真实购物街。

### 本地部署推荐参数

```
Q5_K_M GGUF + 30 steps + dpmpp_2m_sde + sgm_uniform + cfg 6.0
```

或 Q8 GGUF + 30 steps（更锐利）。短边至少 1024px。

## 5. Wan 2.5

### 公式（**Entity + Scene + Motion + Sound**）

```
Entity: [谁/什么 — 详细外观]
Scene: [在哪 — 环境 + 光照]
Motion: [做什么 — 时序动作]
Sound: [听到什么 — voice / sfx / music]
```

### Sound 段是 Wan 的灵魂

支持三种音频元素**同时存在**：

1. **voice/dialogue**：直接给台词，会自动对口型
2. **sfx/sound effects**：环境音、动作音
3. **bgm/music**：背景音乐

例：
```
Sound: Voice: "Welcome back to our weekly update." 
       Professional female voice, room ambient hum, no music.
```

### 数字人 lip-sync 三要素

1. 明确 Entity 是 talking-head（"facing the camera directly"）
2. Motion 写 "lips synced to dialogue"
3. Sound 段给具体台词

样例：[wn-001](../prompts/seedance/README.md) 数字人 lip-sync · [wn-006](../prompts/seedance/README.md) 新闻播报。

## 6. 即梦 3.0

### 8 维度公式（中文友好）

```
[场景] + [人物] + [服装] + [动作] + [镜头] + [光影] + [风格] + [情绪]
```

### 中文写作守则

1. **纯中文优于中英混用**
2. **用中式美学词汇**：江南、汉服、水墨、仙侠、烟雨、苍茫
3. **描述具体到品类**：不写 "美女"，写 "扎马尾的少女"
4. **场景词库**：园林、茶室、戏台、市井、雪山、竹林、星空

### 与剪映集成的工作流

1. 在即梦生成视频素材
2. 一键插入剪映时间线
3. 剪映里加字幕 / 转场 / 音乐
4. 直接发布

适合**单人创作者**完整闭环。

样例：[jm-001](../prompts/seedance/README.md) 江南水乡 · [jm-005](../prompts/seedance/README.md) 仙侠雪山修仙者。

## 6 模型选型决策树

```
你要什么？
├── 编辑已有视频（改光照/换风格/加元素/去人物）  → Runway Aleph
├── 短视频爆款，要"特效感"                       → Pika 2.5（Pikaffects）
├── 极致物理感（液体/火焰/布料/真实动作）         → Hailuo 02
├── 本地部署 / 开源 / LoRA 训练 / 跨维度          → Hunyuan 1.5
├── 数字人对口型 / 播报 / 真人广告               → Wan 2.5
└── 中文剧情 / 中式美学 / 国风                   → 即梦 3.0
```
