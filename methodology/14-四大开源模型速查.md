> 商业模型再强,**开源才是真正的底座** — 本地部署 + LoRA 训练 + 二次开发自由。本仓库收录的 4 个开源/开源友好模型:

|---|---|---|---|---|

## 1. LTX-Video 0.9.7

### 核心 USP

**实时生成是颠覆性的** — 5 秒视频在 5 秒内生成。意味着创作者可以**迭代式实时改 prompt**,而不是等几分钟一次。

### 公式:单段 prose

```
4-8 句单段落自然描述,200 词以内,present tense,literal precise
```

❌ 不要列表 / 不要分段 / 不要 "Style:" 标签
✅ 像电影摄影指导口述分镜一样,自然流动一段话

### 物理 vs 情感

- ❌ "She feels sad" (情感标签)
- ✅ "Her jaw tightens and she looks away" (物理线索)

### 三种独家用法

| 场景 | 写法 |
|---|---|
| **screenplay 对话场景** | 用 `EXT. / INT.` 标准剧本格式(参见 [lt-002](../prompts/data/all-prompts.json)) |
| **图生视频** | 只描述运动,不重述图中已有内容 |
| **everyday 日常** | LTX 擅长写实,**不擅长奇幻夸张** — 别用它跑魔法或赛博朋克 |

### 本地部署

- GitHub: [Lightricks/LTX-Video](https://github.com/Lightricks/LTX-Video)
- HuggingFace: [Lightricks/LTX-Video-0.9.7-dev](https://huggingface.co/Lightricks/LTX-Video-0.9.7-dev)
- ComfyUI 工作流: [ComfyUI_examples/ltxv](https://comfyanonymous.github.io/ComfyUI_examples/ltxv/)
- Modal Cloud 一键部署: [Modal Docs](https://modal.com/docs/examples/ltx)

## 2. Mochi 1

### 核心 USP

**目前已发布的最大开源视频模型** (10B 参数 AsymmDiT)。**强 prompt 遵循** — 你写啥它出啥,几乎不偏。

### 公式:自由文本 + 物理细节

Mochi 没有强结构要求,但**物理细节越精确,效果越准**:

```
A slow-motion shot of a wine glass shattering on a polished concrete floor,
fragments of glass spinning outward through the air, droplets of red wine
arcing in slow trails, captured from a low angle with shallow depth of field.
```

注意:
- 时间词 (slow-motion / time-lapse)
- 物理动作 (shattering / spinning / arcing)
- 摄影词 (low angle / shallow DOF)

### 限制

- **最多 5.4 秒** — 比 LTX 短一半
- 480p 起步版本(高分辨率需更大显存)
- 不擅长长复杂叙事(短场景物理感最强)

### 本地部署

- GitHub: [genmoai/mochi](https://github.com/genmoai/mochi)
- 浏览器试用: [Genmo Playground](https://www.genmo.ai/)
- Replicate API: [genmoai/mochi-1](https://replicate.com/genmoai/mochi-1)

## 3. CogVideoX 5B / 1.5

### 核心 USP

**中国出品 + Apache 2.0 完全开源**(智谱 AI + 清华大学)。**768×1360 高分辨率** + **226 token 长 prompt** + **T2V + I2V 双模式**。

### 公式:长 prompt 详细场景

```
英文最多 226 token (约 170 词),详细描述场景 + 主体 + 动作 + 摄影
```

CogVideoX **欢迎长 prompt** — 越详细越好。这与 LTX(精炼)和 Pika(短)形成对比。

### 双模式

- **T2V (Text-to-Video)**:基础模型
- **I2V (Image-to-Video)**:有专门权重([CogVideoX-5b-I2V](https://huggingface.co/zai-org/CogVideoX-5b-I2V))

### 本地部署

- GitHub: [zai-org/CogVideo](https://github.com/zai-org/CogVideo)
- HuggingFace: [zai-org/CogVideoX-5b](https://huggingface.co/zai-org/CogVideoX-5b) · [I2V](https://huggingface.co/zai-org/CogVideoX-5b-I2V)
- 推荐参数:50 steps + dpmpp_2m + cfg 6.0 + 720p+

## 4. Higgsfield Soul 2.0 / DoP

### 核心 USP

**Soul ID** = 业界最强角色一致性。先用几张图训练你的 Soul ID,之后**所有场景里这个角色脸都不变**。配合 **DoP 镜头模板** + **AI Director 智能分镜**,做 60-90 秒短片是最佳工具。

### 三个看家本领

#### 1. 短指令 > 长描述

❌ "A confident young woman with long brown hair wearing a black leather jacket walks slowly through a neon-lit alley with rain pouring down and reflections on the wet pavement..."

✅ "Slow push in. [Soul ID: jenny] in leather jacket. Neon alley. Rain."

Higgsfield **反常识**:短直接的指令产出更强,长描述反而让它 guess。

#### 2. Soul ID 占位符

```
[Soul ID: my-character] walks into a sunlit kitchen wearing white linen shirt,
opens the fridge, turns to camera with slight smile.
```

`[Soul ID: name]` 替换为你训练过的角色 ID,**所有场景同一张脸**。

#### 3. DoP 镜头模板

直接给运镜动词作为 prompt 起手:

| 模板 | 适用 |
|---|---|
| `Slow push in` | 强调 / 情感 |
| `Whip pan from left to right` | 转场 / 动作 |
| `Crash zoom` | 惊讶 / 揭示 |
| `Crane down` | 全景 / 收尾 |
| `Handheld walk-and-talk` | 纪录 / 对话 |

### AI Director(2026 新功能)

让 AI 帮你分镜 — 你只描述"想要什么样的短片",它产出 60-90s 多场景结构,每场景自动用合适的 DoP 模板。参见 [hg-006](../prompts/data/all-prompts.json) 60s MV 样板和 [hg-008](../prompts/data/all-prompts.json) 90s 短片样板。

- Soul 介绍: [higgsfield.ai/soul-intro](https://higgsfield.ai/soul-intro)
